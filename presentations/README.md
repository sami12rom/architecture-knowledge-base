# Presentations

Outlines and speaker notes for talks, workshops, and internal sessions.

Each talk gets its own folder:

```
presentations/
└── hexagonal-architecture-101/
    ├── outline.md    # slide structure and speaker notes
    └── README.md     # metadata: audience, duration, link to the live deck
```

Don't commit binary slide files (.pptx, .key, .pdf). Link to the live deck instead — the link always points to the current version, and the repo stays lightweight.

## Outline format

```markdown
# Talk Title

**Deck:** [link](URL)
**Duration:** 45 min
**Audience:** Mid-level engineers

---

## Slide 1 — Opening / Problem
- What you're saying on this slide
- Speaker note: ...

## Slide 2 — ...
```

Speaker notes go under each slide heading. Keeping them in the outline means the content is readable and searchable on its own, independent of whatever tool you built the deck in.
